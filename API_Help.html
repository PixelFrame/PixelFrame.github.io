在没有特别说明的情况下，API输入示例为HTTP 1.1 POST，输入示例和响应示例数据格式均为JSON。
在结果为成功的情况下的JSONresult键为success，而失败时为fail并附加错误信息。获得JSON错误表明服务器已正确响应示例，如果发送HTTP错误则表明服务器可能未正确运行。
由于PHP的json_encode函数与弱类型的限制，返回的数据中所有数据均为字符串（Request的Appendix内容除外），在使用前需要进行转换。
[SERVER] = http://47.100.99.193:80/

错误代码 名称 注释

0x02 DATABASE_OPERATION_FAILURE 无法完成指定的数据库操作

0x0A JSON_DECODE_FAILURE 无法解析JSON，发送的JSON数据存在问题或未正确接收到数据
0x0B JSON_FORMAT_ERROR JSON数据可以解析但不符合API的数据格式
0x0C JSON_AUTH_FAILURE JWT认证失败，可能是JWT过期或未正确传递JWT
0x0D JSON_JWT_FORMAT_ERROR JWT格式错误
0x10 USER_NAME_USED 指定的用户名已被使用
0x11 USER_NAME_NOT_FOUND 指定的用户名不存在
0x12 USER_PASSWORD_INCORRECT 密码错误
0x13 USER_PHONE_USED 指定的手机号已注册
0x14 USER_EMAIL_USED 指定的电子邮件已注册
0x15 USER_STDID_USED 指定的学号已注册
0x16 USER_INSUFFICIENT_PERMISSION 用户权限不足
0x17 USER_NOT_FOUND 指定的用户不存在

0x31 REQUEST_NOT_FOUND 请求不存在
0x32 REQUEST_CANNOT_CANCEL 无法取消指定的请求
0x40 INTERFACE_NOT_ACCESSIBLE 此API不可用
